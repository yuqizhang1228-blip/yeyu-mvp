# 左侧抽屉 · 功能范围（冻结）

> **原则**：P0 抽屉 **以当前代码为准** 验收与迭代；对齐 Figma 0515 时 **只改样式，不擅自按 YUQ-30 全文扩功能**。  
> 下文「设计有、代码暂无」**不是产品否决**，而是 **现阶段未做 + 避免 Agent 误加**；要上哪一项，在 Linear 单独立项后再改本文。

## 触发与壳层

| 项 | 实现 |
|----|------|
| 打开 | 首页 / 对话页顶栏左侧菜单；`appState.drawerOpen = true` |
| 关闭 | 点遮罩、选导航项、选历史会话后自动关 |
| 动画 | 自左滑入，宽约 **304pt**，遮罩 `YeyuColor.overlay` |
| 挂载 | `HomeView`、`ChatView` 内各一份 `SideDrawerView`；对话页传入 `currentSessionId` 用于高亮 |

## 面板内容（仅此三块）

### 1. 页眉

- 标题：**夜屿**
- 副文案：`YeyuUser.drawerLabel` → 有昵称显示昵称，否则 **「匿名用户」**（非 Figma 副标题「安放情绪的岛屿」）

### 2. 固定入口（2 个）

| 入口 | 行为 |
|------|------|
| **行动卡片** | 关抽屉 → `openHistory()` → `HistoryView`（已保存卡片列表） |
| **设置** | 关抽屉 → `openSettings()` → `SettingsView` |

### 3. 最近对话

- 标题：**最近对话**
- 数据：SwiftData `ChatSession`，按 `updatedAt` 倒序，最多 **12** 条
- 展示：`session.title`（含 AI 生成标题逻辑，见会话服务）
- 点击：关抽屉；若已在当前会话则 no-op；若在对话页则 `replaceChat(sessionId:)`，否则 `openChat(sessionId:)`
- 当前会话：文案色 `YeyuColor.primary`

## 为什么有一份「设计有、代码暂无」对照？

你提到 **抽屉按当前实际情况保持一致** —— 意思是：**别因为对了 Figma / YUQ-30，就把还没做的入口硬塞进 P0**。

| 设计稿 / YUQ-30 里常见 | 当前 iOS 为何没有 | 以后能不能做 |
|------------------------|-------------------|--------------|
| 会员转化 Banner | P0 不做 IAP；RN 试验稿有占位，SwiftUI 未接 | 可以，需 YUQ-49 / 商业化立项 |
| 个性化 | 有独立 Issue [YUQ-37](https://linear.app/yuqi-design/issue/YUQ-37)，走设置子流，不是抽屉必含 | 可以，按 YUQ-37 做 |
| 支持 / 鼓励 | 产品未定义页面与路由（RN 有「支持」但 iOS 无 `AppRoute`） | 可以，需先定 PRD + 路由 |
| 入口名「心情卡片」 | iOS 已用 **「行动卡片」** 进同一 `HistoryView`，名称不同、能力等价 | 可只改文案对齐设计 |
| 抽屉内新建对话 | **刻意**放在对话顶栏 **+**（与 YUQ-32 一致），避免入口重复 | 一般不必挪回抽屉 |
| 历史左滑删除 | 未排进 P0；清数据在设置 | 可以，单独小票 |

**总结**：不是「这些功能不好」，而是 **P0 只交付已实现的 3 块**；设计轨（YUQ-30）写的是 **目标态**，工程轨（YUQ-45）验收的是 **现实现**。两轨并行时，需要这份对照，免得改 UI 时顺手加一堆未验收能力。
